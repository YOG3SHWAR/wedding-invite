IMPORTANT: Use `frontend-design` skill for UI execution (hard constraint from user).

Create `src/components/sections/rsvp-section.tsx` as a 'use client' component:

**Section wrapper:**
- id="rsvp" for anchor linking
- py-section-mobile md:py-section spacing
- Section heading: "RSVP" in font-heading text-gold, Hindi subtitle "कृपया अपनी उपस्थिति दर्ज करें" in font-hindi

**State management:**
- useState for form fields: name (string), phone (string), guestCount (number, default 1)
- useState<Set<number>> for selectedDays
- useState for status: 'idle' | 'submitting' | 'success' | 'error'
- useState for errorMessage (string)
- When status === 'success', render RsvpConfirmation component instead of form

**"Sabhi Din / All Days" button:**
- Above the day cards
- Large tappable button with gold border
- On click: sets selectedDays to new Set([1, 2, 3])
- If all 3 already selected, deselects all (toggle behavior)

**3 Day cards:**
- Map over RSVP_DAYS constant
- Each card is a large tappable div (min-height ~120px for uncle/aunty fingers)
- Card shows: labelHindi (font-hindi, large), label (font-body), date, event names joined by " + "
- Use inline style={{ borderColor: EVENT_COLORS[day.colorKey] }} for accent color
- Selected state: background fills with accent color (opacity 0.15), border thickens, checkmark icon appears
- Unselected: cream/white background, thin border with accent color
- motion/react whileTap={{ scale: 0.97 }} for tactile feedback
- onClick toggles the day in selectedDays Set

**Form fields (below day cards):**
- Name: large text input, placeholder "आपका नाम / Your Name", font-body, min h-14 for touch
- Phone: tel input, placeholder "फ़ोन नंबर / Phone Number", font-body, min h-14
- Guest Count: number input, min 1, max 20, placeholder "मेहमान / Guests", with +/- stepper buttons on sides for easy mobile use
- All inputs: rounded-lg border border-gold/30 focus:border-gold focus:ring-2 focus:ring-gold/20

**Submit button:**
- Large, full-width, bg-maroon text-white font-heading, min h-14
- Text: "RSVP भेजें / Send RSVP"
- Disabled when: name empty, phone empty, selectedDays empty, or status === 'submitting'
- When submitting: show decorative gold mandala spinner (motion/react rotate animation on a small mandala SVG or decorative circle with animate={{ rotate: 360 }} transition={{ repeat: Infinity, duration: 1.5 }})

**Submit handler:**
- Call validatePhone(phone) -- if invalid, set errorMessage and return
- Call submitRsvp({ name, phone, guestCount, days: Array.from(selectedDays) })
- On success: set status to 'success' (triggers confirmation render)
- On error: set status to 'error', show friendly message "कुछ गलत हो गया / Something went wrong" with a "पुनः प्रयास करें / Try Again" button that resets status to 'idle'

**Error display:**
- Below submit button, text-red-600 with Hindi+English error text
- Retry button styled as outline button

**Wrap sections in ScrollReveal for entrance animations.**

Then update `src/__tests__/rsvp-section.test.tsx` -- replace it.todo() stubs with real tests:

- Submit button is disabled when form is empty
- (Keep complex interaction tests as it.todo if they require extensive async mocking)

**Layout -- full-screen overlay feel:**

- Cream background (bg-cream or the project's background color)
